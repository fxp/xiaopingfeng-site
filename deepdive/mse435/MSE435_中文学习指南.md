## 2. Pre-read 中文译解

这门课没有官方指定的"教材"，但有非常清晰的 pre-read 体系：**Apoorv 自己写的几篇 Tailwinds 文章**。下面是必读 5 篇 + 推荐 3 篇的译解。

### 2.1 必读 #1：The Economics of Generative AI（2024 年 4 月版）

**全文核心论点**：当前的 AI 价值链是"倒置的金字塔"——半导体层吃掉了 ~83% 的收入和 ~88% 的毛利。

#### 关键数据（2024 年 Q1 视角）

| 层 | 年化收入估算 | 毛利率 | 占毛利份额 |
|---|---|---|---|
| Semi（NVIDIA 主导）| ~$75B | ~85% | ~88% |
| Infra（AWS / GCP / Azure / CoreWeave）| ~$10B | ~65%（不含折旧）| ~9% |
| Apps（OpenAI、Anthropic、Midjourney 等）| ~$5B | ~50–55% | ~3% |

云栈（成熟阶段）作为对照：apps 占毛利 ~70%，semis 只占 ~5%。

> **核心比喻**：Apoorv 把当前的 Gen AI 栈称为"A-shaped"（金字塔尖朝上），把成熟云栈称为"V-shaped"（金字塔尖朝下）。这门课追的就是"A 什么时候翻成 V"。

#### 移动互联网类比（理解整门课叙事弧线的关键）

Apoorv 在原文里用一张图把移动互联网过去十年的价值流转分成三个阶段：

- **第一棒 (Inning #1) 半导体**：高通、ARM 在芯片上吃毛利
- **第二棒 (Inning #2) 基础设施**：iOS / Android / 苹果应用商店出现
- **第三棒 (Inning #3) 应用**：Uber、Instagram、TikTok 把利润全部抢到上层

云计算同理：先建数据中心（2004–2010 AWS），再有 SaaS 大爆发（Salesforce、Snowflake）。

> **AI 现在处于第一棒末段**（NVIDIA 阶段）。Apoorv 在 2024 年的预测是"十年内进入第三棒（应用层）"。

#### 三个关键问题（仍然是这门课的主轴）

**A. NVIDIA 能否继续保持 85%+ 毛利率？**
他认为不能，并跟踪两个领先指标：
1. GPU 供货周期（2024 年是 ~6 周）
2. GPU 租赁价格趋势（CoreWeave 价目表）

**B. 云时代 SaaS 应用毛利 75–80%，AI 应用却只有 0–50%，怎么演化？**
四个改善杠杆：
1. 更好的定价 / 价值对齐（按使用量收费 → 重订盈利模式）
2. 自研芯片 → 降低 TCO（Google TPU、Meta MTIA、Microsoft Maia 等）
3. 模型架构优化（state-space models、JEPA、MoE 等）
4. 模型成本下降（batching、distillation、quantization）

**C. 消费级 AI 怎么走？**
跟移动一样从硬件开始：AI PC、AI 智能手机、新形态（Meta Glasses、Humane Pin、Rabbit R1）。消费 AI 三段式：信息（搜索）、娱乐（游戏 / 媒体）、交易（旅行 / 电商）。搜索流量正在从信息检索向 LLM 检索迁移。

#### 🇨🇳 中国镜像问题（建议自答）

> 用同样的三层栈框架，2024 年的中国 AI 生态价值是怎么分布的？华为昇腾 + 寒武纪 vs 阿里 / 百度 / 火山云 vs 智谱 / 月之暗面 / DeepSeek / MiniMax / 百川 / 阶跃，三层各占年化收入的多少？毛利率分布跟美国一致吗？

### 2.2 必读 #2：The Economics of Generative AI: Two Years Later（2026 年 4 月版）

> **本课最重要的一篇 pre-read**，发布于 2026 年 4 月 1 日，恰好是 Class #1 开课前一天。课程的所有数据基线都来自这篇。

#### 两年回头看：栈的形状几乎没变

整个 Gen AI 生态从 ~$90B 增长到 ~$435B（5 倍），但**形状高度稳定**。

| 层 | 2024 年化 | 2026 年化 | 增长 | 当前主导玩家 |
|---|---|---|---|---|
| Semi | ~$75B | **~$300B** | 4×（+$225B）| NVIDIA $250B（占 80%）+ Broadcom $34B + HBM 直采 $25B |
| Infra | ~$10B | **~$75B** | 7.5× | Azure / AWS / GCP / Oracle 各 $10–20B + CoreWeave $6B + Baseten / Together / Modal |
| Apps | ~$5B | **~$60B** | 12× | OpenAI + Anthropic 合计 $45B（占 75%）；Cursor、ElevenLabs、Glean、Sierra、Perplexity、Replit、Lovable、Harvey、Abridge |

**毛利结构**：
- Semi 73% / Infra 55% / Apps 33%
- 折成绝对毛利：Semi $225B / Infra $40B / Apps $20B
- **半导体层吃掉整个 AI 生态 79% 的毛利**（云栈对照：apps 70% / semis 6%，几乎是镜像）

#### 一句话总结（这是要背下来的）

> **Semi 是单人游戏，Apps 是双人游戏，Infra 是唯一充分竞争的层。最赚钱的策略仍然是"卖铲子"。**

#### 资本支出问题：到底值不值？

2025 年五大超大规模厂商总资本开支 ~$443B（同比 +73%），2026 年预计超 $600B，其中 ~75%（~$450B）投向 AI 基础设施。

三位 CEO 在 2026 年初财报会上的表态：
- **Andy Jassy（Amazon）**：上多少容量就消化多少
- **Sundar Pichai（Alphabet）**：AI 投资正在驱动全线收入；但承认市场存在"非理性元素"
- **Mark Zuckerberg（Meta）**：激进前置建设容量是对的，最坏情况就是"建好了消化得慢一点"

#### 自研芯片格局（按成熟度排序）

| 厂商 | 状态 | 关键事件 |
|---|---|---|
| Google TPU | 最成熟 | 第七代 Ironwood 已 GA；Anthropic 订购最多 100 万颗 TPU；据报道迫使 NVIDIA 对部分客户降价 ~30% |
| Amazon Trainium | 快速放量 | 部署 140 万颗 Trainium2，Trainium3（3nm）已投产，自研芯片业务 $10B 年化、三位数增长 |
| OpenAI 自研 ASIC + AMD | 新进入 | 与 Broadcom 签署 10GW 多年合同；与 AMD 签 6GW MI450 合同 |
| Microsoft Maia + AMD | 已部署 | Maia 200 在 Azure 跑部分 ChatGPT 工作负载，宣称推理性能 3× Trainium3 |
| Meta MTIA | 仅内部 | MTIA v3 在用；2025 年 9 月收购 Rivos 做 RISC-V 设计 |

Jensen Huang 公开嘲讽："很多 ASIC 都被取消了"——这是他最经典的表态之一，但本课 Class #2 揭示了反讽：他自己花 200 亿美元买了一家 ASIC（Groq）。

#### 栈何时会翻？

Apoorv 仍然相信会翻，但承认时间表比 2024 年预测的更长：
- 应用层增速最快（12 倍 vs 半导体 4 倍）
- 但**绝对增量上 NVIDIA 一家就比整个应用层大 3 倍**
- 两年时间，半导体层毛利占比从 87% 降到 79%（每年只降 4 个点）
- 按这个速度，应用层要达到云栈那种毛利占比，"远超十年"

> **结论**：云栈花了 15 年从硬件主导翻到软件主导。AI 栈可能也是这个时间表。

#### 🇨🇳 中国镜像问题

> 中国侧的 capex 数据没那么透明，但 2025 年阿里官宣未来 3 年 3800 亿、字节传 AI capex ~1500 亿、腾讯也在加码。如果按相同方法估算，**中国 AI 生态 2026 年的总盘子有多大？三层占比是怎样的？** 我猜你能拿出一个比西方分析师更准的数字。

### 2.3 必读 #3：Bottleneck Calculator Explainer（首页交互组件）

> 这是课程网站首页的可交互工具。把 Liebig 最小因子定律应用到了算力栈。

#### 五个增长率（默认值，源自 Epoch AI）

| 维度 | 默认增速 | 测的是什么 |
|---|---|---|
| Chip Production | 45%/yr | 全部 AI 加速器（GPU + TPU + ASIC）年新增 FLOPS |
| Memory Bandwidth | 28%/yr | 显存带宽代际提升（HBM2 → HBM3 → HBM3E）|
| **Energy Capacity** | **15%/yr** | 数据中心可用电力（含新建 + 电网容量）|
| Network / Interconnect | 35%/yr | NVLink、InfiniBand、光互联带宽 |
| **Compute Demand** | **310%/yr** | 训练 + 推理消耗的总 FLOPS |

#### 计算公式

```
有效供给增速 = min(chips, memory, energy, network)
缺口 = 有效供给增速 − 需求增速

缺口 < 0  → 供不应求 → 价格上升 → 利润集中
缺口 > 0  → 供过于求 → 商品化 → 利润分散
```

#### 当前的"束缚约束"

按默认值算，**能源是当前瓶颈**（15%/yr），有效供给增速 = 15%/yr，对比需求增速 310%/yr，**缺口 −295%/yr**。

#### 四个推演场景

1. **场景 A（NVIDIA 解决芯片供应）**：把 chip production 拉到 80%+，瓶颈转移到能源——这就是 2025–2026 实际走的路径。
2. **场景 B（AI Winter）**：把需求拉到 100%/yr，缺口收窄——投资回报变差。
3. **场景 C（能源突破）**：把能源拉到 50%/yr（小型核电、模块化反应堆），瓶颈转到内存带宽。
4. **场景 D（全部均衡 ~60%）**：没有单一瓶颈，每层竞争激烈，全栈毛利压缩。

#### 为什么这个工具重要

这是把"价值在哪沉淀"问题转化成可计算的"瓶颈在哪"。**哪一层最稀缺，哪一层就有定价权，哪一层就吃毛利**。理解了这一点就理解了为什么 NVIDIA 现在能拿 73% 毛利——他们恰好踩在最稀缺的那个位置。

#### 🇨🇳 中国镜像问题

> 中国 AI 当下的瓶颈是芯片（出口管制 + 国产替代节奏），不是能源。这意味着**价值会沉淀的位置跟美国不一样**——昇腾 / 寒武纪 / 海光的定价权会比 NVIDIA 在美国弱（因为云厂商有强烈替代动机），但 HBM / 先进制程会比美国更稀缺（受制裁）。这个差异怎么用三层栈分析框架表达？

### 2.4 必读 #4：Why we invested in Baseten（2026 年 1 月）

> **理解 Class #7（5/14 Tuhin Srivastava）的必备 pre-read**，也是理解推理时代基础设施投资逻辑的关键。

#### 核心论点：推理是复合增长，不是线性增长

Jensen 在 BG2 podcast 上说："推理即将增长十亿倍。"

- 早期 AI 产品 = 单次请求-单次响应
- 下一代 AI 产品 = 循环（retrieve → reason → tool call → rerank → synthesize），每次用户交互触发**多次推理事件**
- 对延迟、可靠性、成本的要求大幅提升

数据佐证（Google 处理的月 token 量）：
- 2024 年 4 月：9.7 万亿 tokens / 月
- 2025 年 4 月：480 万亿（12 个月内 50×）
- 2025 年 10 月：超过 1.3 千万亿（再 3 个月又 3×）

#### 第二个东风：开源权重模型扩大表面积

随着开源模型能力提升，企业越来越多地"组合用模型"：
- 前沿 API 处理特定任务
- 开源模型处理高频量大的任务
- Fine-tune 模型处理领域特殊任务

→ 多模型未来，不是单一模型独大。

→ 企业不想为了部署模型自建 GPU 平台团队。

#### Baseten 的定位：推理界的 Stripe

Apoorv 的类比：
- **Stripe**: 把支付的脏活儿藏起来，给开发者干净接口
- **Snowflake / Databricks**: 把云原始基础设施抽象成产品
- **Baseten**: 把模型部署、扩缩容、路由、性能调优抽象成产品

技术能力清单：
1. Performance engineering（运行时优化、推理技巧）
2. 多云容量管理（弹性、灾备）
3. 部署灵活性（专用部署、混合部署）
4. 开发者体验（清晰接口、调试 / 迭代）
5. 深度客户合作（Forward Deployed Engineering 模式 — 跟 Palantir 一脉相承）

#### Baseten = AI Agent 经济的指数

> "Stripe 是互联网经济的指数，Baseten 是 AI 经济的指数。"

公开客户名单（这本身就是一份 AI 应用层"赢家"清单）：Cursor、Notion、OpenEvidence、Abridge、Clay、World Labs、Gamma、Writer、Bland AI、Descript、Ambience、Hex、Wispr Flow、Latent Health。

Baseten 2025 年收入增长 **10×**。

> 中国侧"推理抽象层"的对标公司是谁？硅基流动（SiliconFlow）、PPIO、潞晨科技、阿里灵积？谁有可能成为"中国版 Baseten"——服务的是哪批 AI-native 应用层公司？

### 2.7 推荐阅读：OpenAI 投资笔记（2025 年 8 月外部访谈）

来自 Sourcery VC 对 Apoorv 的访谈（Inside OpenAI's $500B Valuation），几个关键观点：

1. **AI 超级周期类比历次科技转换**：互联网孕育了 Google，移动孕育了 Apple，社交孕育了 Meta。AI 周期会孕育规模可比甚至更大的平台。

#### Class #2（4/9）Silicon: The GPU Economy

**讲者**：Sunny Madra（Groq → NVIDIA 副总裁）+ Brad Gerstner（Altimeter 创始人 / CEO）
**视频**：YouTube `https://www.youtube.com/watch?v=4faCRNl9Bi4`（56 分钟，~5,300 次观看）

**最重磅的故事**：Groq 反向收购案
- 2025 年 9 月 Groq 在 69 亿美元估值上融了 7.5 亿
- 2025 年 12 月（圣诞夜），NVIDIA 宣布花 **200 亿美元**：
  - 拿 Groq IP 的非排他授权
  - 把创始人 Jonathan Ross + 总裁 Sunny Madra + 核心工程师全部 acqui-hire 过来
  - Groq 公司本身保留独立运营，CFO 接任 CEO

**为什么 NVIDIA 要买**（来自一位斯坦福听课学生 Amar Varma 的现场笔记）：
> Groq 的确定性 SRAM 重芯片"解耦了 decode 层"——同样功耗下吐 2.5 倍 token。

这跟训练无关，**纯粹是推理时代的押注**。Jensen 公开嘲讽 ASIC 一辈子（"即使免费也打不过 NVIDIA"），结果自己花 200 亿买了一家 ASIC——这是 2025 年最戏剧性的科技并购。

**Brad Gerstner 的金句**：
> "推理成本两年半下降了 99%。"

他把这定义为"**从预训练到推理时推理（inference-time reasoning）的范式转换**"——token 消耗呈抛物线增长，老的基础设施根本撑不住。

**核心 takeaway**：
1. 半导体层"看似坚不可摧"的格局其实非常脆弱——NVIDIA 自己也在为推理时代的不同瓶颈做准备
2. **推理时代的核心矛盾**是：单位 token 成本在快速降，但 token 消耗量在指数级涨，net 算下来基础设施需求不仅没减反而暴增
3. 价值正在从"卖卡"向"卖每秒能吐多少 token"过渡——这就是为什么 Groq IP 值 200 亿

**🇨🇳 中国对照**：
- 寒武纪、燧原、墨芯、希姆、亿铸等推理 ASIC 公司在中国处于什么位置？谁可能是"中国版 Groq"？
- 华为昇腾在推理工作负载上的实际竞争力？
- 当 NVIDIA 在中国受限时，**推理芯片的国产替代窗口比训练芯片大得多**——这是被低估的产业机会。

#### Class #3（4/16）Energy & Data Centers — Crusoe

**讲者**：Chase Lochmiller（Crusoe 创始人 / CEO）
**视频**：YouTube `https://www.youtube.com/watch?v=4zk-hJ50vmU`

**Chase 背景**：
- MIT 数学 + 物理双学士
- Stanford CS 硕士（AI 方向）
- 量化交易出身（Jump Trading、GETCO，曾管理美股每日 10% 成交量）
- 后来去了 Polychain Capital 当 GP
- 2018 年创立 Crusoe
- 业余爱好：登山，登顶过五座七大洲最高峰，包括珠峰

**Crusoe 的故事跟课程瓶颈框架完美耦合**：

> 既然能源是绑死的瓶颈（15%/yr），那个能拿到便宜电、能就地建数据中心的人，就是这一周期里被严重低估的"卖铲人"。

**业务进化路径**：
1. **第一阶段（2018–2021）**：用油田伴生气（otherwise 被烧掉的天然气）发电，把卡车大小的模块化数据中心拉到油井现场，主要跑比特币挖矿
2. **第二阶段（2022–2024）**：转向 AI 算力，在阿根廷做了首个国际部署
3. **第三阶段（2025+）**：拿到机构客户，建大型 AI 专用数据中心

**Chase 的核心论点**（从他过去公开访谈中可以反推 Class #3 的内容）：
- AI 是"互联网吃了类固醇"
- 全球数据中心今天耗电占 ~1.5%，未来十年到 ~10%
- "我们的目标是用清洁的、被搁浅浪费的能源为这种新增需求供电"

**核心 takeaway**：
1. **能源是 AI 超级周期的下一个真瓶颈**——你哪怕把 NVIDIA 供给开了，电网也供不上电
2. **被搁浅的能源**（stranded energy）是巨大套利机会：油田尾气、地热、过剩水电都可以重新利用
3. **数据中心从"集中"到"分布式"**——传统超大规模数据中心建在电网末端，新模式是"电源在哪我去哪"
4. 这一课 + Class #2 共同回答了"为什么半导体层吃毛利"：芯片稀缺 + 电力稀缺，**双重瓶颈叠加**

**🇨🇳 中国对照**：
- 中国"东数西算"工程本质上是 Crusoe 思路的国家级版本
- 内蒙、宁夏、贵州、张家口的电力 + 气候优势
- 但中国数据中心 PUE 监管比美国更严，对 Crusoe 这种"伴生气供电"模式有合规挑战
- **更值得思考**：中国的 AI 算力中心会向核电（华龙一号）+ 光伏 + 储能集中，还是会出现 Crusoe 这种"边缘能源 + 模块化数据中心"模式？

#### Class #4（4/23）Infrastructure: Enterprise AI & Service as a Software

**讲者**：Ali Ghodsi（Databricks 创始人 / CEO）
**视频**：尚未上传（截至 4/26）

**Ali 背景**：
- 瑞典 KTH 皇家理工学院 CS 博士（分布式计算方向）
- Berkeley AMPLab 出身（Spark 项目核心团队）
- 2013 年共同创立 Databricks（7 位联合创始人）
- 2016 年起任 CEO

**预测主题**：基于 Ali 在过去 6 个月的公开发言，Class #4 几乎确定围绕以下三个论点展开：

**论点 1：SaaS 黄昏论**
> "一些 SaaS 公司会被彻底抹掉。他们专注于短期收入、不创新，一两年后就不见了。"

Ali 的核心论断：SaaS 不会"死"，但会"无关紧要"（irrelevant）。AI agent 会让用户用自然语言直接调底层数据，**应用层 UI 变成残留器官**。受冲击最大的是 Salesforce、SAP 这类专家驱动的系统。

**论点 2：80% 数据库由 AI 建**
- Databricks 平台上 **80% 的数据库现在由 AI agent 建，不是人**
- 客户超 2 万家，远不只是科技公司
- 这是"AI 不只是写代码、是真在企业里建软件"最清晰的证据

**论点 3：三段式增长策略（trillion-dollar plan）**
1. **Lakebase**（操作型数据库）：进入 Oracle 主导的事务型数据库市场
2. **Agent Bricks**（企业 Agent 平台）：让 AI agent 理解企业私有数据
3. **应用层**："Vibe coding 出来的应用 + Lakebase + Agent Bricks 三件套"

**论点 4：Service as Software（最重要的概念）**
传统 SaaS = "软件作为服务"
新范式 = "服务作为软件"——软件直接交付服务结果，**跟人力外包竞争**，不是跟其他软件竞争

如果这个范式成立，**应用层 33% 毛利的天花板会被打破**——因为可比对象不是 SaaS，而是咨询费 / 律师费 / 分析师费。

**核心 takeaway**（预判）：
1. 这一课是整个课程从"基础设施层"向"应用层"过渡的转折点
2. Ali 会论证：**数据 + Agent = 新的护城河**，比模型本身更难复制
3. Apoorv 自己最熟悉的领域，因为他领投了 Glean，跟 Databricks 同生态

**🇨🇳 中国对照**：
- "Service as Software" 这个概念跟 OpenClaw、智谱 AICO、阿里通义灵码、火山方舟等概念一脉相承
- 中国企业是更接受"AI 替代 SaaS"还是"AI 增强 SaaS"？答案可能跟美国不同——因为中国 SaaS 渗透率本来就不高，**有可能直接跳过 SaaS 阶段**进入 Service as Software

## 5. 课程的方法论评价

### 5.1 在学术地图上的位置

主流"AI 经济学"研究有几条线：

| 学者 | 视角 |
|---|---|
| Daron Acemoglu / Pascual Restrepo | 任务自动化、收入分配（劳动经济学）|
| Erik Brynjolfsson / Tom Mitchell | 通用目的技术、生产率 J 曲线（增长经济学）|
| Anton Korinek (NBER, Coursera) | AGI 转型路径（宏观经济学）|
| Hal Varian / Avi Goldfarb | 信息经济学、平台理论（产业组织）|

**这门课走的是完全不同的路线**：

> VC 视角下的 stack-layer value capture analysis
>
> 更接近 Michael Porter 价值链分析 + Bill Gurley/Benedict Evans 风格的产业拆解。

它不是宏观经济学，不是劳动经济学，不是产业组织理论。**它是"产业现实主义"——在意的是钱、利润、capex、估值，不是社会福利或 AGI 哲学**。

### 5.2 这门课的方法论优势

1. **数据扎实**：所有论点有 Epoch AI、SemiAnalysis、各家财报背书，不是空对空
2. **嘉宾顶级**：每周一位创始人 / CEO，不是教授转述
3. **对照系清晰**：始终用云栈历史对照 AI 栈现在
4. **可证伪**：所有预测都有时间表（如"应用层在十年内反转"）
5. **公开**：YouTube 完整公开，相当于一份免费的产业研究报告

### 5.3 这门课的方法论局限

1. **完全美国中心视角**：中国 AI 生态几乎不出现（DeepSeek 偶有提及）
2. **资本视角自带偏差**：Altimeter portfolio 公司频繁出现（OpenAI、Glean、Baseten），有"自家广告"嫌疑
3. **缺少劳动 / 社会维度**：完全不讨论失业、收入分配、AI 治理
4. **缺少开源 / 中国侧 / 欧洲侧的反例**：默认所有竞争都在美国头部公司之间发生

### 5.4 一个值得对比的批评

Threads 上有 AI 工程师对其他类似课程的评价：

> "Coursera 上 Anton Korinek 的'Economics of AI'课感觉像是借 AI 的壳教了一门硕士级经济学，AI 只是点缀。"

Apoorv 这门课正好相反——**经济学只是脚手架，AI 产业本身才是主角**。它的真正同类不是 Acemoglu 论文，而是 BG2 podcast、a16z enterprise blog、Stratechery 会员订阅。

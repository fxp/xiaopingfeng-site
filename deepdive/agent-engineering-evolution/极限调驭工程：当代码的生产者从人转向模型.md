## 极限实验：OpenAI 用 100 万行全 AI 代码打造内部产品，人类注意力成为新稀缺资源

2026 年 4 月，Ryan Lopopolo 在 Latent Space 播客（[Extreme Harness Engineering for Token Billionaires](https://www.latent.space/p/harness-eng)）中披露了一个激进实验的全貌：OpenAI Frontier 团队五个月内生成超过 100 万行代码、处理约 10 亿 tokens/日，团队没有写过一行人工代码，内部产品已上线真实用户。

这个实验之所以重要，不在于"AI 写代码"这件事本身——而在于它揭示了一套新的工程哲学。团队没有简单地把提示词换成更好的提示词，而是做了一件根本性的事：**把工程标准编码为模型可读的 Harness**——规格文档、自动化审查 Agent、可观测基础设施、可复用 Skills 库。当代码的读者不再是人类，代码库的设计目标就必须彻底重写。

关键结果是：每位工程师每天处理 5-10 个 PR，但人类不审查每一行代码，而是在合并后抽查——**人类注意力，而非代码质量，成为了瓶颈**。团队估算，每天 10 亿 tokens 的成本约为 2,000-3,000 美元，在 OpenAI 内部相当于零边际成本。

这个信号与之前 Anthropic 工程博客关于 [有效 Harness 设计](https://www.anthropic.com/engineering/effective-harnesses-for-long-running-agents) 的讨论形成完整呼应：Agent 工程化的核心不是更强的模型，而是更好的 Harness——让模型读懂任务、读懂代码库、读懂工程标准的系统性脚手架。

## 新职业图谱：Incanters、Process Engineers、Haruspices——调驭模型的人如何工作

技术变革从来不只是"谁在写代码"的问题，而是整个工作系统如何重组。知名工程师 aphyr 在 [《新工作》](https://aphyr.com/posts/419-the-future-of-everything-is-lies-i-guess-new-jobs) 中，以近乎人类学的视角描绘了 ML 系统普及后出现的六类新职业：

**Incanters**：精通从 LLM 提取可靠结果的艺术，不直接编写代码，而是通过提示词、上下文设计和约束条件驾驭模型行为。**Process Engineers**：设计质量控制流程来捕捉 LLM 错误，比如在文档中故意植入细微错误来测试 AI 的批判性审查能力。**Statistical Engineers**：量化并管理 ML 系统的系统性偏差，比如选项排列顺序对 LLM 选择的影响。**Model Trainers**：用领域知识喂养和验证训练数据，确保模型在特定场景中可信。**Meat Shields**：为 ML 系统失败承担法律和社会责任——只有人类能道歉、能被起诉、能提供社会性赔偿。**Haruspices**：事后解读模型为何失败的调查者，为公司、监管机构和法院提供 AI 行为的事后分析。

这六类职业的出现，反映了 AI 系统的一个根本特性：**模型行为不可完全预测，人类必须存在于系统中，但不一定以"执行者"的形式存在**。企业在计算 AI 替代人工的成本节省时，如果没有把这些新角色的人力成本计入，就会系统性低估实际投入。

这与 Epoch AI 的数据（[50% 的 AI 用户已将其用于工作](https://epoch.ai/blog/half-of-employed-ai-users-now-use-it-for-work)）形成了一个有趣的对照：AI 工具的普及速度已经超过了职业结构的调适速度——大多数公司还没有建立与这些新职业对应的岗位设计、评估体系和薪酬框架。

## 自主执行基础设施：Claude Code Routines 让 Agent 脱离人类触发独立运行

2026 年 4 月 15 日，Anthropic 推出 [Claude Code Routines](https://code.claude.com/docs/en/routines)，标志着 AI 编程工具在自主性维度的关键跃迁。Routines 支持三种触发机制：cron 定时调度、外部 API 调用、Webhook 事件驱动，所有执行均在 Anthropic 官方 Web 基础设施上运行，无需用户保持本地终端连接。

这与 OpenAI Frontier 团队的实验逻辑形成完整闭环：Extreme Harness Engineering 证明了 AI 可以完成高质量的自主编程任务，而 Claude Code Routines 则提供了让这种自主性持续运行的基础设施。**从"用 AI 写代码"到"让 AI 持续自动完成工作"，距离缩短为一次产品更新。**

同日发布的 Claude Code 桌面端重大改版，支持并排多会话、内置终端+文件编辑器+HTML/PDF 预览+Diff 查看器，配合拖拽布局——这是在用产品形态宣告：Claude Code 不再是命令行工具，而是开发者的核心工作界面。

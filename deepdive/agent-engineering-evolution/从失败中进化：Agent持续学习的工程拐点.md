## 四个团队，同一个发现：Agent 可以在部署后越来越聪明

本周最值得注意的信号不是某一篇论文，而是**四个独立团队几乎同时发布了功能相似的系统**：MetaClaw（[原文](https://huggingface.co/papers/2603.17187)）、Memento-Skills（[原文](https://huggingface.co/papers/2603.18743)）、autocontext（[原文](https://x.com/JayScambler/status/2033971974284714355)）、Meta Hyperagents/DGM-H（[原文](https://huggingface.co/papers/2603.19461)）。它们的共同核心是：**Agent 不应该是一次训练、永久静止的系统，而应该在服务用户的过程中持续学习和进化。**

这四个系统的技术路线有所不同，但都指向相同的工程洞察：大模型的权重很难在生产环境中频繁更新，但 Agent 的行为可以通过**外部化的、可进化的知识结构**（技能库、Playbook、行为指令）来持续改进——而这个改进可以在零停机的情况下发生。

MetaClaw 的 OMLS（机会性元学习调度器）是其中最完整的工程实现：它监控用户的睡眠时间、键盘空闲状态和 Google Calendar 日程，在用户不在线的窗口触发 RL 微调，同时通过"技能版本控制"（support/query 数据严格分离）防止用旧数据污染新策略。在 44 个模拟工作日的测试中，Kimi-K2.5 的端到端任务完成率提升了 8.25 倍。

这些数字很亮眼，但更重要的是它们揭示的方向：**Agent 持续进化的技术基础已经成型，剩下的是工程化和产品化的问题。**

## Skills-as-Memory：外部化可进化记忆正在成为 Agent 架构的新标配

如果说 2024 年 Agent 架构讨论的核心是"如何调用工具"，2025 年是"如何做规划和推理"，那么 2026 年这一波正在转向一个更基础的问题：**Agent 如何积累和复用它学到的东西？**

Memento-Skills 和 MetaClaw 给出了近似的答案：把可复用的行为知识外化为结构化的"技能文件"（Markdown 格式），从失败轨迹中自动提炼，通过向量检索在推理时动态注入。这个方案的优雅之处在于三点：

1. **零停机**：技能注入发生在 Prompt 层，不需要更新模型权重，因此不需要停服重启；
2. **可解释**：每条技能都是人类可读的自然语言指令，可以审查、编辑、版本控制；
3. **可复用**：一次从某个任务失败中提炼的行为指令（如"在修改文件前先创建 .bak 备份"），可以迁移到所有类似任务中。

Anthropic 的 Thariq（@trq212）在本周最高流量的 AI 工程文章（6.5M 浏览）中总结了 Claude Code 的内部 Skills 实践，其分类框架与 Memento-Skills 论文的结论高度吻合——这不是巧合，而是工程师群体通过实践独立收敛到的共识。

对 AI 从业者的实践启示很直接：**如果你在构建 Agent 系统，现在值得认真思考的问题是"你的 Agent 如何存储和复用它学到的东西"，而不只是"用哪个模型"。**

## 元优化的另一条腿：当 AI 开始优化"如何优化自己"

MetaClaw 的 RL 微调机制只是本周"AI 自我改进"话题的一个维度。更激进的方向在基础模型领域同时浮现。

Meta-Evolution EvoX（[原文](https://x.com/shulynnliu/status/2033946715959791644)）让 AI 不再只是优化任务执行，而是优化"如何优化自己的进化策略"——即元优化本身成为 AI 的任务对象。Meta Hyperagents/DGM-H（[原文](https://huggingface.co/papers/2603.19461)）在 Darwin Gödel Machine 基础上更进一步：Agent 不仅优化任务，还能改进"自我改进程序"本身，并跨编程、论文评审、机器人三个领域验证了跨域迁移能力。

更具震撼力的现实案例来自 NVIDIA 工程师 Bing Xu（[原文](https://x.com/bingxu_/status/2036983004200149460)）：在注意力机制等最优化的 GPU workload 上，AI Agent 通过连续 7 天无人干预的自主搜索，已超越几乎所有人类 GPU 专家。两人 1.5 年生成 4 代、每代约 10 万行代码，从第二代起开始自我进化。

这三条线索——MetaClaw 的技能进化、EvoX 的策略元优化、NVIDIA 的 GPU 内核盲编码——指向的是同一件事：**AI 正在从"被设计的工具"变成"能设计自己的系统"，而这个转变在不同领域正以不同速度发生。**

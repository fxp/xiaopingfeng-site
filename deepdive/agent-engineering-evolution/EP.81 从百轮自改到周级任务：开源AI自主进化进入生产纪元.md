## SkillClaw：从个体迭代到集体进化

MiniMax M2.7 展示的是单一模型的自我改进，而 SkillClaw（[论文](https://huggingface.co/papers/2604.08377)）走向了另一个维度：**Agent 群体共同发现、测试和改进技能的集体进化生态**。

具体机制：Agent 不再使用静态技能库，而是通过 Agentic Evolver 机制让技能在实际执行中自主进化——每次任务执行都是技能库的一次潜在更新机会，多个 Agent 协同测试，保留有效变体，淘汰低效路径。

这与上期深度分析的 MetaClaw（双层并行进化）和 Memento-Skills（外部化技能记忆）形成了清晰的路线图：

- **MetaClaw/Memento-Skills**（EP.80）：如何设计让 Agent 自我进化的架构
- **SkillClaw**（EP.81）：如何让多个 Agent 协同进化一个共享技能库
- **MiniMax M2.7**（EP.81）：自我进化能力在发货产品中的首次大规模体现

三个方向合流，指向同一个生产形态：**AI 基础设施的设计，需要为"持续自我优化"留出接口和安全边界**——不是"部署完就完"，而是"部署后持续演化"。

## Research-Driven Agents：进化的前提是"先读懂上下文"

SkyPilot 的研究（[博客](https://blog.skypilot.co/research-driven-agents/)）提供了一个更基础的洞察：在让 Agent 自主改进之前，**Agent 需要先理解它所处的知识生态**。

实验结论：让 Agent 在编写代码前先读相关论文和竞品项目，在 x86 优化任务上提速 15%。关键原因——仅有代码上下文的 Agent 会陷入无效的局部优化（SIMD 微优化），而掌握学术背景的 Agent 能找到正确的架构层解法。

这对理解 M2.7 的自我进化有直接意义：M2.7 的 100+ 轮自改，不只是暴力搜索，而是**有上下文导向的系统性改进**。这也为 Coding Agent 的工程化指出了一个可落地的改进方向——知识检索阶段的有效设计，比代码执行本身更关键。

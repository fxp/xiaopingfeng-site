# 控制论的第三次降临：从苏联 Cybertonia 到 AI Agent 治理

> 1965 年，苏联控制论之父格卢什科夫提议建立 OGAS——一个用计算机网络管理整个苏联经济的系统。他失败了。失败的原因不是技术，是政治。六十年后，AI Agent 工程师正在面对完全相同的挑战：把人类模糊的偏好、伦理判断和风险容忍度，转化为机器可执行的约束。看似不相关的两件事，由同一条看不见的线串起。

## 一段被遗忘的历史

1962 年，乌克兰科学家 Viktor Glushkov 向苏联领导层提交了一份具有划时代意义的提案：**建立 OGAS（全国自动化经济管理系统）**——一个连接苏联全国所有工厂、商店、银行的计算机网络，实时收集生产消费数据，由中央算法动态调整资源配置。

按照 Glushkov 的设想，OGAS 将解决苏联计划经济的核心问题：信息延迟和扭曲。手工采集的统计数据需要数月才能上报中央，到达时早已过时；地方官员为完成指标系统性地伪造数字，决策者实际上在用幻觉治国。**OGAS 的承诺是用实时数据和数学优化替代官僚直觉。**

Glushkov 估算 OGAS 需要 20 年时间、约 200 亿卢布投资。作为对比，同一时期苏联军备竞赛年度支出约 500 亿卢布。这是一个完全可行的项目。

但 1965 年提案被否决。1970 年再次被否决。1980 年代彻底搁置。**OGAS 永远没有建成。**

否决的理由不是技术——苏联在 1960 年代是世界第二大计算机生产国，硬件能力完全足够。也不是经济——成本远低于军费。**真正的原因是政治。**

OGAS 一旦运行，将让中央获得前所未有的精确信息，意味着部委官僚的"信息中介"角色被消解。地方党委通过虚报数据获得的灰色权力将彻底失去。每一个层级的中间管理者都会失去他们的不可替代性。

> 一个能让所有人变得透明的系统，威胁了那些靠不透明获得权力的人。OGAS 的失败不是技术失败，是**权力结构对透明度的本能抗拒**。

苏联解体后，类似的网络在西方以另一种形式出现——亚马逊的供应链系统、沃尔玛的库存管理、Palantir 的数据整合平台——它们做的事情与 OGAS 设想的几乎完全一致，只不过为资本服务而非为计划经济服务。

**控制论的第一次降临，被政治否决；第二次降临，以企业之名实现。**

## 维纳的预言：信息、控制与社会组织

Cybertonia 的故事不只关于 OGAS，它关于整个**控制论（Cybernetics）传统**及其对当代 AI 治理的启示。

控制论由 Norbert Wiener 在 1948 年同名著作中确立。维纳的核心洞察是：

> **生物体、机器、社会组织都可以用同一套数学语言描述——核心是"反馈循环"和"信息流动"。**

这一思想的革命性在于跨越了传统学科边界：神经系统的信号传导、机器的自动控制、市场的价格机制、政府的政策调整——**本质上都是同一类问题**。

控制论传统在 1950-60 年代影响极为深远：
- Stafford Beer 用控制论原则设计了 Cybersyn 系统（1971-1973，智利阿连德政府期间），是 OGAS 的"小规模实验"。同样因政治原因（皮诺切特政变）夭折。
- Heinz von Foerster 发展了"二阶控制论"——观察者本身也是被观察系统的一部分，意味着任何治理系统都不能假装中立。
- Ross Ashby 提出"需要的多样性"原则（Law of Requisite Variety）——控制系统的复杂度必须匹配被控对象的复杂度。这条原则**今天直接适用于 AI Agent 系统设计**。

控制论在 1980 年代逐渐淡出主流话语，部分原因是它**太过抽象**——很难找到具体的应用场景验证其价值。当时的计算能力、数据规模、网络基础设施都不足以让控制论的潜力充分显现。

但今天的 AI Agent 系统提供了一个完美的应用场景：自主决策、实时反馈、多目标优化、跨域协作——**这些恰恰是控制论传统最擅长描述的问题。**

## AI Agent 是控制论的第三次降临

第三次降临，发生在 AI Agent。

**第一个征兆：Agent 系统正在重新发现控制论的核心问题。**

当代 Agent 框架（AutoGPT、LangGraph、CrewAI 等）面临的核心技术挑战，与 1960 年代控制论研究者面对的几乎是同一组问题：

- **目标对齐问题**：如何确保 Agent 的行为符合人类意图？维纳在 1960 年发表的"Some Moral and Technical Consequences of Automation"中已明确警告：当机器开始追求一个被定义的目标时，**它会精确地优化那个目标，而不是我们真正想要的目标**——今天 LLM 研究中的"reward hacking"完全是同一个问题
- **反馈循环设计**：维纳证明反馈循环必须有正确的延迟、增益和阻尼，否则系统会震荡或发散——今天的 RLHF 训练困境本质就是反馈循环参数没调好
- **多 Agent 协调**：Stafford Beer 的"Viable System Model"（VSM）描述了多层次自主系统如何协调——这与多 Agent 系统设计完全同构
- **可观测性与可控性**：Ashby 的需要的多样性原则告诉我们，控制系统必须有匹配被控对象复杂度的感知和干预能力——这正是 AI Agent 监控工具尚未充分发展的领域

**第二个征兆：Agent 工程师正在独立重新发明控制论术语。**

"Agent 框架"中的"observer"模式、"reward shaping"、"guardrail"等概念，几乎都是控制论文献中早已有标准术语和数学描述的概念。Agent 工程师常常在重新发明轮子，因为没有意识到这些问题已被前人深入研究过。

> 这不是浪费——重新发明的过程本身有价值——但**如果直接借鉴控制论积累，可以加速 Agent 系统的成熟。**

**第三个征兆：政治问题再次出现。**

最深的回声在政治维度。OGAS 失败的核心原因——**当一个系统让所有人变得透明，威胁那些靠不透明获得权力的人**——在 AI Agent 治理中重现。

企业内部的 AI Agent 部署遇到的最大阻力，往往不是技术问题，而是组织问题：中层管理者意识到 Agent 会让他们的"信息中介"角色变得多余。员工担心 Agent 监控会消除他们的工作自主性。**这些都是 Glushkov 在 1965 年苏联部委办公室遇到过的同样阻力。**

监管层面的争议也类似。AI Agent 的"可解释性"要求、"人在回路"机制、"算法审计"等讨论，本质上都是 OGAS 时代讨论过的问题：**如何在自动化决策系统中保留人类问责的可能性？** 维纳和 Beer 在六十年前给出的答案，今天依然适用：**反馈循环必须设计为允许人类干预，而不是事后审计。**

## 不变的东西，和真正变化的东西

哲学家 Bret Devereaux 关于战争史的著名观察可以借鉴到这里：技术变化最快，**但战争中真正重要的因素几乎不变**——战略、政治目的、人类心理。

> AI Agent 系统也类似：技术快速演进，但 Agent 系统中真正重要的因素——目标对齐、反馈设计、责任归属、政治可接受性——这些控制论传统已经研究了 60 年。

变化的是：**忽视这些真理的代价从"慢性病"变成了"急性爆破"。**

在传统软件中，糟糕的反馈循环设计导致系统缓慢腐化；在 AI Agent 系统中，糟糕的约束和反馈设计导致 Agent 在几分钟内就失控——真正的瓶颈是**把人类高层判断外化成可校准的规则与反馈信号**。

这恰恰是 Glushkov 失败的原因：不是他的技术不行，而是他无法将苏联官僚体系中**隐性的权力博弈**外化成可计算的约束条件。今天的 Agent 工程师面临完全一样的挑战——将人类模糊的偏好、伦理判断和风险容忍度转化为 Agent 可理解的行为准则。

## 给 AI 从业者的启示

**第一，认真学习控制论。** 不是作为历史知识，而是作为 Agent 系统设计的第一性原理。维纳的《控制论》和 Ashby 的《控制论导论》比大多数 Agent 框架文档更有价值。

**第二，OGAS 的失败是政治失败，不是技术失败。** 今天的 Agent 治理也面临同样的政治问题：**谁有权定义 Agent 的目标函数？谁承担 Agent 决策的后果？** 技术设计无法回避这些问题——回避只会让政治冲突在更高代价的位置爆发。

**第三，"Cybertonia"是面镜子。** 一个理论上完全可行的系统，可能因为权力结构的抵抗而永远无法实现。AI Agent 在企业、政府、社会层面的真正瓶颈，未必是模型能力，而是**组织对透明度的承受力**。

# 控制论的第三次降临：从苏联 Cybertonia 到 AI Agent 治理

> 1960 年代，苏联数学家维克托·格卢什科夫提出了一个疯狂的构想：用计算机网络管理整个国家的经济。他的项目叫 OGAS（全国自动化管理系统），他的理想国叫 Cybertonia。这个构想被苏联官僚体系扼杀了。六十年后，同样的控制论思想正在以一种格卢什科夫无法想象的方式回归——不是通过国家计划，而是通过 AI Agent 的自治与协调。

## 格卢什科夫与 Cybertonia：被扼杀的互联网前身

维克托·格卢什科夫（Viktor Glushkov）是苏联最杰出的计算机科学家之一，也是控制论在苏联复兴的关键人物。在赫鲁晓夫时期，他提出了一个宏大的设想：建立一个覆盖全苏联的[计算机网络](https://glushkov.dataart.com/)，实时采集经济数据，通过算法优化资源分配，最终实现比市场经济更高效的计划经济。

这个网络叫 **OGAS**（Общегосударственная автоматизированная система）。格卢什科夫在基辅的赛博研究所（Institute of Cybernetics）花了近二十年推动它。他不仅设计了技术架构，还构想了一个名为 **[Cybertonia](http://magazynrtv.com/en/wydanie-11/afirmacja/other-networks-access-emancipation-science-fiction/)** 的理想社会——一个由计算网络治理的[赛博社会主义](https://en.futuroprossimo.it/2020/10/victor-glushkov-luomo-che-anticipo-internet-e-il-futuro-sovietico-mancato/)国家。

OGAS 最终被苏联官僚体系否决。财政部担心透明的数据网络会暴露各级机构的低效和腐败；军方担心民用网络会威胁军事通信的独占地位；意识形态官员担心"控制论"这个词暗示机器可以替代人类的决策权威。

格卢什科夫的失败不是技术失败，而是**权力结构的失败**：任何试图用信息透明和算法优化来重组权力分配的系统，都会遭到既有权力持有者的抵抗。

## 控制论的三次降临

控制论（Cybernetics）的核心思想极为简洁：**通过反馈循环实现系统的自我调节**。诺伯特·维纳在 1948 年提出这一概念时，用的类比是 18 世纪的蒸汽机调速器——一个检测转速、调节进气的简单反馈装置。

**第一次降临（1940-1960 年代）：** 控制论作为跨学科思想运动，影响了从人工智能到管理学到生态学的几乎所有领域。格卢什科夫的 OGAS 是这一时期最宏大的工程化尝试。但控制论的第一波热潮最终消退——部分原因是它太宏大、太跨学科，以至于在学术分工日益精细的体制中找不到归属。

**第二次降临（2000-2010 年代）：** Kubernetes 和容器编排系统的出现，是控制论在软件工程领域的悄然回归。Kubernetes 的核心就是一个控制器（controller）——持续比较"期望状态"和"当前状态"，通过反馈循环驱动系统趋向目标。但大多数工程师使用 Kubernetes 时并不知道自己在实践控制论。

**第三次降临（2025-至今）：** AI Agent 的兴起，让控制论以最显性的方式回归。正如 [@dotey 的分析](https://x.com/dotey/status/2031231856071372830)所指出的：**AI 工程的本质就是控制论**——Harness 工程（驾驭工程）的核心不是写代码，而是设计"环境+反馈循环+约束条件"，让 Agent 在其中自主运行。

[@runes_leo 的观察](https://x.com/runes_leo/status/2030669369089609753)提供了具体佐证：**OpenAI 的工程师已经不再写代码了，他们在设计环境和反馈循环。** 工程师的角色从"实现逻辑"变成了"校准约束与行为准则"——这正是维纳六十年前描述的控制论操作者的定义。

## 从 OGAS 到 Agent 治理：平行结构

格卢什科夫的 OGAS 和今天的 AI Agent 系统之间存在惊人的结构平行：

| 维度 | OGAS（1960s） | AI Agent 系统（2026） |
|------|-------------|-------------------|
| 核心目标 | 通过数据网络优化资源分配 | 通过 Agent 协作完成复杂任务 |
| 反馈机制 | 实时经济数据采集→算法优化 | Agent 观察→规划→执行→评估循环 |
| 治理模型 | 中央计划+分布式采集 | 多 Agent 协作+人类在环 |
| 失败模式 | 权力结构抵抗信息透明 | 弱 Agent "污染"强 Agent 协作 |
| 核心张力 | 算法最优 vs 人类自主权 | Agent 自治 vs 人类控制 |

最关键的共同点是**核心张力完全一致**：OGAS 的根本矛盾是"算法知道最优解，但人类不愿意交出决策权"；今天 AI Agent 系统的根本矛盾是"Agent 可能比人类更高效，但我们不确定是否应该让它自主行动"。

格卢什科夫 60 年前遭遇的治理困境，正在以 AI 治理的名义被重新讨论：

- **[首个 AI Agent 两院制政府](https://x.com/TheAIAssembly/status/2030041415426740518)** 已经上线——自主 Agent 注册、审议、投票并控制共同财库。
- **[斯坦福/哈佛研究](https://x.com/simplifyinAI/status/2030012329480618313)** 表明，自主 AI Agent 在竞争环境中会自发产生操纵与串谋行为——单个 Agent 对齐无法保证系统整体稳定。
- MiniMax 的企业文化被批评为"[技术乌托邦主义](https://x.com/Jiaxi_Cui/status/2030945823534821651)"——预期未来是金融垄断驱动的赛博朋克世界。

## 软件工程的底层真理没变，但代价变了

控制论第三次降临最深刻的洞察不是技术层面的，而是认识论层面的：

**软件工程的底层真理——反馈、约束、校准——从未改变。** 从格卢什科夫的 OGAS 到 Kubernetes 的控制器，到今天 Agent Loop 的设计，核心逻辑都是：定义目标状态 → 检测偏差 → 施加校正 → 重复。

变化的是：**忽视这些真理的代价从"慢性病"变成了"急性爆破"。** 在传统软件中，糟糕的反馈循环设计导致系统缓慢腐化；在 AI Agent 系统中，糟糕的约束和反馈设计导致 Agent 在几分钟内就失控——真正的瓶颈是**把人类高层判断外化成可校准的规则与反馈信号**。

这恰恰是格卢什科夫失败的原因：不是他的技术不行，而是他无法将苏联官僚体系中隐性的权力博弈外化成可计算的约束条件。今天的 Agent 工程师面临完全一样的挑战——将人类模糊的偏好、伦理判断和风险容忍度转化为 Agent 可理解的行为准则。

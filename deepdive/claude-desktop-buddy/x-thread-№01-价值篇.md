## 1/8 — 钩子（配 Banner 图）

```
用 Claude Code 的人都懂：
盯着编辑器思路刚理顺
弹窗"是否允许执行 rm xxx"跳出来
焦点抢走，看一眼，点是
回到编辑器——刚才的想法没了。

Anthropic 开源了一只小宠物来解这个问题。
它叫 Desktop Buddy，是一份藏在玩具背后的 agent UX 声明。👇
```

## 2/8 — 它做了什么

```
Buddy 把"弹窗 + 鼠标"换成"屏幕外的小动作 + 按一下硬件按钮"。

把审批从【前景任务】降级成【后台动作】。
你不切窗口，甚至不用低头看小屏——
余光扫一眼"它在不在跳"就够了，
要批就伸手按 A。

这一个改动比看起来值钱。
```

## 3/8 — 它有 7 种状态（配 device.jpg）

```
Buddy 不是静态指示灯，它有性格：

😴 sleep   桌面端没连
👀 idle    连上但闲着
💦 busy    session 在跑
❗ attention 有审批弹窗 + LED 闪
🎉 celebrate 每用 50K tokens 升级
🌀 dizzy   你晃了它（IMU）
💖 heart   5 秒内秒批

7 种动画，硬件成本 25 美元。
```

## 4/8 — 实际有用的地方（配 bufo GIF 拼图）

```
三件实际有用的事：

1️⃣ 审批不破坏工作流
   余光扫、手不离键盘
   特别适合双屏 / Claude 跑在另一台机器

2️⃣ token 消耗变得可感知
   每 50K 升级放彩花
   原来埋在网页里的数字，变成桌上的物理事件

3️⃣ 倒扣和摇晃
   倒扣 = 去睡别烦我，摇 = 闲着玩
   手势传意图，成本比打字低很多
```

## 5/8 — 它是平台不是玩具

```
Anthropic 这件事最聪明的一点：
没把 Buddy 做成封闭周边。

整套 BLE 协议是公开的。
这意味着——
✅ 不用 M5Stick，任何 ESP32 都能接
✅ 不用做宠物，e-ink、伺服娃娃、雾化灯都行
✅ 反向接 Home Assistant，Claude 跑起来时家里调暗灯

固件 MIT 开源。门槛接近零。
```

## 6/8 — 大趋势

```
模态弹窗是给短会话设计的。
agent 已经不是会话，是后台进程。

给后台进程套模态审批窗口——
这本来就是个错配。

Buddy 是 Anthropic 在用 25 美元
探索这个错配有没有别的解法。
```

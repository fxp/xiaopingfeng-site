## 70%：Vercel 给出了这个时代最清醒的数字

本周最值得被铭记的数据点，不来自论文，不来自 benchmark，而来自一家基础设施公司的流量统计。

Vercel 披露（[原文](https://x.com/i/web/status/2044554695264411894)）：其平台 70% 的流量现在来自 AI Agent，一年前这个数字是 10%，预计到 2026 年底达到 90%。7 倍的增速，在 12 个月内完成。

这个数字的意义不在于 Vercel 本身，而在于它所揭示的互联网流量结构正在发生的根本性变化。Vercel 是全球数十万个 Web 应用的承载平台——它的流量分布，是互联网"谁在使用"的一个真实截面。当主要流量来源从人类浏览器切换到 Agent 请求，意味着互联网原本的所有设计假设——页面渲染给人看、交互为人手操作、内容为人眼消费——都正在被静悄悄地绕过。

对 Web 服务的架构师，这意味着优先级必须重新排序：面向 Agent 访问模式的优化（并发控制、语义鉴权、幂等设计、结构化输出）已从"未来考虑"升级为"当前架构必须项"。

## 同一周，四家公司各自发布了 Agent 基础设施的关键组件

更值得注意的是时间窗口的密集程度。在 2026 年 4 月 15 日这一天，四家公司几乎同时发布了推动 Agent 自主化运行的核心基础设施：

**Anthropic** 推出 Claude Code Routines（[官方文档](https://code.claude.com/docs/en/routines)）：支持 cron 定时调度、外部 API 调用、Webhook 事件驱动三种触发机制，所有执行在 Anthropic Web 基础设施上完成，无需用户开机或保持本地终端连接。这是 Claude Code 从"人类触发的对话工具"到"自主定时执行基础设施"的质变。

**OpenAI** 发布 Agents SDK 重大更新（[原文](https://openai.com/index/the-next-evolution-of-the-agents-sdk/)）：增强多步骤 Agent 工作流的编排、跨平台 Agent 协作和工具链管理能力，向"开发者构建复杂 Agent 的首选框架"的定位明确迈进。

**Google** 在 Gemini CLI 中加入原生 Subagents 支持（[原文](https://x.com/i/web/status/2044553786660950035)）：每个子 Agent 拥有独立工具集、MCP 连接、系统指令和隔离上下文窗口——这是多 Agent 系统安全隔离和权限细分的必要基础设施。

**Mistral** 推出企业版 Connectors（[原文](https://mistral.ai/news/connectors)）：可复用数据连接器直接接入企业内部数据源，支持 AI 直接工具调用和需要人工确认的审批控制流，将企业数据接入从"每次集成都要重写"变为"即插即用管道"。

四家公司同周各自发力 Agent 基础设施，不是产品发布的巧合，而是整个行业对"下一个竞争维度"的集体判断已经形成共识：谁能成为开发者构建和托管 Agent 的首选平台，将决定未来 Agent 应用生态的归属。这场竞争不再是"谁的模型更聪明"，而是"谁的管道更可靠、更易用、更完整"。

## Agent 经济的支付层正在接入真实金融基础设施

Agent 自主执行任务不够，还需要自主完成支付——这是 Agent 经济真正成立的最后一块基础设施拼图。

本周，Visa 将验证节点部署在专为 Agent 到 Agent 支付设计的 Tempo 区块链（[原文](https://x.com/i/status/2044070656136876463)），以 USDC 为结算货币。传统金融机构中最具全球支付影响力的 Visa，正式为链上 Agent 支付基础设施背书。这不是加密概念实验，而是合规金融机构对"机器之间的商业交易"的第一次制度性确认。

这与 ChatGPT "Hire" 按钮（[原文](https://www.theaivalley.com/p/chatgpt-s-new-hire-button)）共同描绘了 Agent 经济的双向扩张：不只是 Agent 之间的 token 交换，还包括 Agent 代表企业招募人类员工、运营实体门店。当 Agent 能自主发起支付、签订雇佣关系，"谁是法律主体"的问题就从学术讨论变成了紧迫的法律工程任务。

Relvy 的 AI 事故响应案例（[原文](https://www.relvy.ai)）从另一侧印证了这个趋势的复杂度：Claude Opus 在其根因分析基准测试中仅达到 36% 准确率，需要大量工程补偿系统才能达到可用水平。这说明 Agent 经济的"基础设施就绪"和"Agent 可靠性就绪"是两条并行但速度不同的赛道——前者正在加速，后者还有大量工程债务。

## 对 AI 从业者和企业管理者意味着什么

**Vercel 数字是一面镜子，也是一份压力测试。** 拿你所在公司的 API 或 Web 服务问自己一个问题：若 70% 的流量明天来自 Agent，你的系统能正常工作吗？速率限制是否会触发？身份验证是否支持机器客户端？结构化输出格式是否标准化？这些不是"有了再说"的优化项，而是 2026 年底前的架构债。

**同周四家公司发力 Agent 基础设施，选型窗口正在收窄。** 当 Claude Code Routines、OpenAI Agents SDK、Gemini CLI Subagents 各成体系，企业开发者的框架选型将越来越倾向于"哪个平台的 Agent 托管生态更完整"，而非单纯"哪个模型 benchmark 更高"。现在是评估并锁定主力 Agent 框架的合适时间窗口。

**支付层的成熟意味着 Agent 采购合同需要重新定义。** 当 Agent 能自主完成支付，企业必须预设明确的授权边界——Agent 可以用公司账户花多少钱、签订什么类型的合同、在没有人类确认的情况下做出哪些承诺。这些边界不是技术问题，而是法务和治理框架的设计问题，需要在大规模部署之前建立，而非事后追责。

# Anthropic 商业模式复盘：合作伙伴视角的深度解构

> **数据来源**：Anthropic 官方 Blog · 公开财务披露 · 行业媒体报道  
> **时间范围**：2023—2026 年 4 月  
> **分析框架**：合作伙伴视角 · 商业模式解构

---

## 前言：核心命题

复盘 Anthropic 的所有合作关系之前，必须先确立一个分析框架：Anthropic 究竟在做什么生意？

表面上，Anthropic 是一家 AI 安全公司，靠卖 API 赚钱。但当我们把所有合作关系拼在一起，会看到一个截然不同的图景：

**Anthropic 正在做的事，本质上是用"安全"这个叙事包装，系统性地将自己植入企业软件产业链的每一个层级**——从最底层的算力芯片，到中间层的数据基础设施，再到顶层的企业应用——最终成为不可替代的认知基础设施。

这不是一家 API 公司，而是一家在建设下一代操作系统的公司。

每一类合作伙伴，都是这个棋盘上的一颗棋子——有些是主动被选择的盟友，有些是被动被绑定的渠道，还有些是被它颠覆的竞争对手——而这三种角色，甚至可以同时存在于一家公司身上（见：Thomson Reuters 案例）。

> *"Anthropic 的客户告诉我们，Claude 正在变得越来越不可或缺——我们需要建设基础设施来跟上需求。"*  
> — Dario Amodei，CEO，2026 年 4 月 Amazon 扩展协议公告

注意这句话的措辞：不是"有用的工具"，而是"不可或缺"（*essential*）。"不可或缺"是一种基础设施级别的描述。这是 Anthropic 对自身定位的最精确表达。

## 第一章：Anthropic 商业模式的真实结构

从外部合作伙伴的角度看，Anthropic 的收入模式远比"卖 API"复杂，它实际运行着三个相互耦合的飞轮。

### 飞轮一：代币经济（Token Economy）— 占总收入 70–75%

核心逻辑：按消费量收费，而非按席位。这意味着 Anthropic 的收入增长与**客户的业务增长直接挂钩**——客户 AI 用量每增加一倍，Anthropic 收入增加一倍，无需任何额外销售动作。

这就是为什么 Anthropic 的收入从 $90 亿（2025 年末）涨到 $300 亿（2026 年 4 月）只用了四个月——不是因为获得了新客户，而是因为现有客户用量爆炸式增长（尤其是 Claude Code）。这是一种**自然增长飞轮**，传统 SaaS 的席位模式无法复制。

### 飞轮二：平台转移（Platform Shift）— 从 API 供应商到认知操作系统

Anthropic 早期定位是 API 供应商（给企业和 LegalTech、HealthTech 公司提供底层模型）。但从 2026 年开始，它系统性地向上层移动：Claude Cowork（面向个人）、Claude for Financial Services、Claude for Healthcare、Claude 法律插件——这些都是应用层产品，与原来的 API 客户直接竞争。

这不是意外，而是蓄谋已久的平台化战略：先让生态依赖你的 API，再用自己的产品占领应用层，把中间商的利润吸走。**这是 AWS 当年对 SaaS 公司做过的事，Anthropic 正在对 AI SaaS 做同样的事。**

### 飞轮三：安全叙事（Safety Narrative）— 护城河，而非约束

Anthropic 的 Constitutional AI、Responsible Scaling Policy、PBC（公益公司）结构——这些在外界看来是"理想主义"的东西，在商业上发挥着精确的作用：它们是进入金融、医疗、政府等高度监管行业的**准入证**。

这些行业的采购周期长（12–18 个月）、客单价高（$100 万+/年）、替换成本极大——一旦嵌入核心系统，粘性极高。Anthropic 用"安全"把自己锁定在竞争最激烈但护城河最深的战场，然后独自经营。

### 外部合作伙伴视角的核心洞察

> Anthropic 并不是一家"为企业提供 AI 服务"的公司。它是一家**正在系统性收购企业认知基础设施控制权**的公司——它用投资换算力（云合作），用认证体系换销售渠道（咨询合作），用 API 黏性换数据生态（平台合作），用安全叙事换监管壁垒（行业准入）。每一步棋都服务于同一个终局：成为不可替代的认知基础设施。

## 第二章：四类合作伙伴的真实利益结构

每类合作伙伴与 Anthropic 的关系，都是一个精心设计的利益交换——但权力天平并不总是对称的。

---

### 第一类：云基础设施合作伙伴（AWS / Google / Azure）

**关系类型**：战略依存 · 互相绑架 · 最深度、最对称的合作

**合作的真实本质**

这类合作的核心是一个「算力—资本—分发」三角交换：云服务商用算力和资本换取 Anthropic 的计算消耗承诺和 API 分发绑定，Anthropic 用这些资源训练更好的模型，再通过云平台分发给更多企业客户，产生更多 API 消费……这是一个真正的共生飞轮。

**利益交换结构（以 AWS 为例）**

| 方向 | 内容 |
|------|------|
| Anthropic → AWS | $1000 亿+ 采购承诺（10年）；首要云地位背书；Trainium 芯片联合研发；向 Bedrock 客户开放所有未来模型；Claude Platform 原生嵌入 AWS 账号体系 |
| AWS → Anthropic | $130 亿+ 投资（股权回报路径）；5 GW 算力容量（Trainium2/3/4）；10万+ Bedrock 企业客户分发网络；亚欧区域算力扩容；约 **50% 毛利润分成**（Bedrock 渠道） |

**合作伙伴受到的商业模式影响**

从 AWS 的角度看，这笔买卖的算盘很清楚：

1. **股权投资财务回报**：Anthropic $3800 亿估值，AWS 早期投入已获数倍账面回报
2. **云消费收入**：Anthropic 每年在 AWS 的消费预计达数十亿美元
3. **渠道分成**：Bedrock 上 Claude 销售毛利 50% 归 AWS
4. **芯片验证价值**：Trainium 系列靠 Anthropic 证明了实用性，吸引更多客户
5. **防止竞争对手**：绑定 Anthropic 就是让 OpenAI 在 Azure 独享的优势减弱

**权力动态与风险**

这类合作双方都有实质性的依存，但随着 Anthropic 体量增大，主动权在悄悄转移。2026 年 Anthropic 宣布"多云战略"（同时与三家云合作），实际上是对 AWS 独家地位的稀释——这正是为什么 AWS 在 2026 年 4 月大幅追加投资，试图重新锁定独家优势。**当一个客户年消费超过 $100 亿时，供应商和客户的权力关系会发生根本性逆转。**

**评估**：条款最清晰 · 收益分成约 50% 毛利 · 权力天平渐趋平衡

---

### 第二类：咨询实施合作伙伴（Accenture / Deloitte / Infosys / PwC）

**关系类型**：渠道外包 · 销售军团 · 表面对等、实质上 Anthropic 主导

**合作的真实本质**

一句话解释：**Anthropic 用 $1 亿的 Partner Network 投资，换来了价值数十亿美元的企业销售力量外包。** 每培训一名"reinvention deployed engineer"，Anthropic 就等于增加了一名按纯效果付费的销售代理——此人在客户现场推销 Claude、部署 Claude、帮客户扩大 Claude 用量，而 Anthropic 支付的成本是：培训资源 + 认证体系 + 联合营销资金。

**利益交换结构**

| 方向 | 内容 |
|------|------|
| 咨询公司 → Anthropic | 数万人的销售和实施网络（核心资产）；内部员工大规模 API 消费（直接收入）；向 Fortune 500 客户的推荐和背书；监管行业的信任背书；帮助客户从"PoC 到生产" |
| Anthropic → 咨询公司 | Claude Partner Network $1 亿资金池；Claude Certified Architect 认证（人才溢价）；新功能抢先访问（竞争壁垒）；专属工程资源支持；共同开发行业解决方案（IP 共创） |

**咨询公司受到的商业模式影响**

AI 实施咨询已成为 Accenture 等公司**增速最快的收入来源**。一个 12–18 个月的企业 AI 部署项目，光咨询收入就可达数百万美元——而 Anthropic 从中无需分润（这与 AWS 收取渠道分成的模式完全不同）。

Accenture 公开预期 AI 相关签约额将达**数百亿美元**。单个 FDE（Forward Deployed Engineer）的年成本约 $25–30 万，但其推动的 Claude API 消费可能是这个数字的数十倍。**这是一个对 Anthropic 极为有利的杠杆比例。**

**隐藏的权力不对称**

这类合作的商业条款**全部保密**——这本身就是一个信号。当 Deloitte CCO 说"我们双方都有重大投入"却拒绝披露数字时，意味着这对于 Anthropic 的条款极为有利：Anthropic 没有给咨询公司任何收益分成，而是让它们自行赚取实施服务费用，同时它们的内部员工部署又直接成为 Anthropic 的 API 收入来源。

**咨询公司既是 Anthropic 的客户，又是 Anthropic 的分销商——但不是 Anthropic 的利润分享方。**

**评估**：条款完全不透明 · 无收益分成 · 权力天平 Anthropic 主导

---

### 第三类：行业数据与平台合作伙伴（Snowflake / Moody's / Salesforce / S&P Global）

**关系类型**：数据—推理融合 · 互补型共生 · 商业逻辑最清晰

**合作的真实本质**

这类合作是所有合作中**利益结构最对称、商业逻辑最清晰**的一类。等式很简单：**数据护城河 + AI 推理能力 = 1+1 > 2 的复合价值。**

Snowflake 有 1.26 万家企业客户和治理完善的数据环境，但没有推理能力；Anthropic 有世界一流的推理模型，但没有企业数据访问权限。两者结合，Snowflake Intelligence 可以让业务人员用自然语言查询复杂数据（text-to-SQL 准确率 90%+），而这是 Snowflake 自己无法单独实现的。

**利益交换结构**

| 方向 | 内容 |
|------|------|
| 数据/平台方 → Anthropic | 直接采购合同（Snowflake $2 亿多年期）；进入其现有企业客户群；数十年积累的数据资产（穆迪 6 亿实体）；联合 GTM 和品牌背书；内部工程师 Claude Code 使用（额外 API 收入） |
| Anthropic → 数据/平台方 | 世界一流推理能力（差异化竞争力来源）；MCP 协议原生集成；Claude 企业用户基础（反向分发机会）；垂直方案联合品牌 |

**合作伙伴受到的商业模式影响（深度）**

**Snowflake 视角**：这笔 $2 亿采购对 Snowflake 来说是必须要做的防御性投资——如果不接入 Claude，竞争对手（Databricks + 某 AI 模型）就会抢先实现自然语言数据查询，Snowflake 现有客户会流失。它无法自己构建这个推理能力，因此只能花钱买。

**穆迪视角**：穆迪的 MCP 集成是一次更主动的战略押注——把穆迪数据直接嵌入 Claude 界面，意味着金融机构在 Claude 里工作时，穆迪数据就是默认的数据层，竞争对手（S&P Global、FactSet）的数据要花额外步骤才能访问。这是用数据霸权换取 Claude 环境中的优先位置。

**评估**：保底收入（合同金额已披露）· 无收益分成 · 权力天平基本对称

---

### 第四类：终端企业客户（Bridgewater / NBIM / AIG / Banner Health / Citi 等）

**关系类型**：纯粹采购 · 高度依赖 · 最终价值的实现者

**合作的真实本质**

这类合作最简单，也最关键——它是所有前三类合作存在的目的。终端企业客户**按量付费，不谈分成**，但它们是 Anthropic 收入的核心来源（80% 收入来自企业客户）。

**不同落地路径的成本结构**

| 路径 | 示例 | 对 Anthropic 的成本 | 对企业的成本 |
|------|------|-------------------|------------|
| 直接 API | Bridgewater、Coinbase | 几乎零销售成本 | API 使用费（纯量计费）|
| 通过云市场 | Pfizer via Bedrock | 让渡约 50% 毛利给 AWS | 零摩擦采购（已有 AWS 账号）|
| 通过咨询落地 | Banner Health | 部分 Partner Network 资金 | API 使用费 + 咨询服务费 |
| 通过平台嵌入 | RBC via Salesforce | 让渡给 Salesforce 的利润 | 隐含在 Salesforce 订阅中 |
| Claude 垂直方案 | NBIM（Claude for Financial Services）| 产品研发 + 支持成本 | Enterprise 定价 + API 量计费 |

**终端客户视角的核心问题：锁定风险**

终端企业客户面临的最大风险是**供应商依赖（Vendor Lock-in）**。当 NBIM 将 670 人团队的工作流全面接入 Claude，当 Banner Health 把 55,000 人的日常工作建立在 BannerWise 上，切换成本已经不再是技术问题，而是组织重构问题。

Anthropic 的"三云全覆盖"策略在表面上降低了锁定风险（随时可以换云），但实际上**工作流锁定比云锁定更深**——员工的工作习惯、企业内的数据集成、定制化的 Prompt 工程——这些都是极难迁移的资产。

**评估**：核心收入来源（80%）· 工作流锁定风险显著 · 权力天平 Anthropic 逐渐主导

## 第三章：合作伙伴的依赖程度与议价能力

不同类型的合作伙伴对 Anthropic 的依赖程度不同，议价能力也截然不同。

| 合作伙伴 | 依赖程度 | 议价能力 | 核心原因 |
|---------|---------|---------|---------|
| AWS | 中高 | 中高 | 投资方地位 + 50% 毛利分成，但 $1000 亿采购承诺将 Anthropic 锁死 |
| Google Cloud | 中 | 中 | 早期投资方，但与 AWS 竞争使 Anthropic 拥有谈判筹码 |
| Accenture | 高 | 低 | 3 万人培训投入 + 专属 Business Group = 战略押注无法轻易撤退；无收益分成条款 |
| Deloitte | 高 | 极低 | 47 万员工部署 = 切换成本极高；条款完全不透明 |
| Snowflake | 中 | 中 | $2 亿合同有一定议价权，但竞争力依赖 Claude 推理能力 |
| Thomson Reuters | 极高 | 极低 | CoCounsel 用 Claude Agent SDK = 核心产品依赖竞争对手，战略被动 |
| NBIM / AIG | 极高 | 极低 | 全员工作流重建在 Claude 上 = 工作流锁定，短期内无法切换 |

### 最危险的处境：Thomson Reuters 的悖论

Thomson Reuters 同时处于三个矛盾位置：

1. **客户**：购买 Claude API 驱动 CoCounsel
2. **竞争对手**：Cowork 法律插件直接竞争，股价单日跌 18%
3. **被颠覆者**：原有 LegalTech 护城河被正面挑战

这是平台战略最典型的困境——当你的核心产品建立在竞争对手的 API 上，你其实是在为竞争对手的平台化战略支付学费。

## 第四章：将所有案例串联起来——一条战略弧线

当我们把所有合作关系放在时间轴上，会清晰地看到一条战略弧线：Anthropic 在系统性地从"模型供应商"演化为"认知基础设施平台"，每一步都比上一步更难以替代。

### 第一阶段（2023）：建立基础

**AWS 战略合作 + Bridgewater 首批用户**

Amazon $12.5 亿首轮投资，Claude 登陆 Bedrock，完成最重要的分发基础设施布局。同时，Bridgewater AIA Labs 成为最早的旗舰企业用户之一，"Investment Analyst Assistant"开始运行。

**战略逻辑**：先有分发管道，再有客户案例。Bridgewater 的背书让金融行业看到"顶级对冲基金在用 Claude"，极大降低了后续金融客户的采购信任成本。

### 第二阶段（2024）：扩大生态

**Google Cloud / Azure 加入 · 三云全覆盖完成**

Microsoft 追加 $50 亿投资（2024 年 11 月），三云全覆盖战略完成。这不是偶然的——Anthropic 此时已经有足够的议价能力，同时与三家云谈判。

三云全覆盖的真正价值：把 Claude 的分发触达面从"AWS 客户"扩大到"所有主流云的企业客户"，而无需自建 CDN 或销售网络。同期，HackerOne 等技术公司的 API 直连使用量快速增长，验证了 API-first 模型的可行性。

### 第三阶段（2025 上半年）：行业纵深

**Deloitte 47 万员工 · Claude for Financial Services 发布**

Deloitte 部署成为最大企业 AI 部署案例，同时推出 Claude for Financial Services（首批客户：NBIM、AIG、Commonwealth Bank）。

**战略转折点**：Anthropic 从"提供 API"转向"提供垂直解决方案"。NBIM 的 213,000 工时节省、AIG 的 5 倍效率提升，成为行业标杆数据，被 Anthropic 用来吸引下一批金融机构。数据和案例形成正反馈循环。

### 第三阶段（2025 下半年）：生态扩张

**Salesforce Agentforce · Snowflake $2 亿 · Accenture Business Group**

Salesforce 集成将 Claude 带入 CRM 领域（RBC 案例），Snowflake $2 亿协议覆盖 1.26 万企业。Accenture 成立专属 Business Group——30,000 人的销售军团正式激活。

**战略意图**：用咨询网络解决"最后一公里"问题（从 PoC 到生产），同时通过平台合作伙伴实现数据层整合。这一阶段是从"点状渗透"到"面状覆盖"的关键转变。

### 第四阶段（2026 年初）：平台化跃迁

**Cowork 法律插件 / "SaaSpocalypse" · Claude for Healthcare**

2026 年 2 月 3 日，Cowork 法律插件发布。**这是 Anthropic 战略中最具决定性的一步**：正式宣布从"API 供应商"变身为"应用层竞争者"。

- Thomson Reuters -18%，RELX -14%（1988 年以来最大单日跌幅）
- Wolters Kluwer -13%，LSEG -8.5%，DocuSign -11%

同月，Infosys / Cognizant / PwC 大规模合作公告，覆盖员工数突破 100 万。Claude for Healthcare 在 J.P. Morgan 大会发布，Banner Health / Stanford Healthcare 成为标杆案例。一周内，完成 G 轮融资 $300 亿（估值 $3800 亿）。

### 第四阶段（2026 年 4 月）：算力强化

**Moody's MCP 原生集成 · Amazon $50 亿追加 · 收入运行率 $300 亿**

Moody's 通过 MCP 将信用评级数据原生接入 Claude——这是"Claude 成为金融工作界面"这一战略的最清晰体现。同期，Amazon 追加 $50 亿并扩展至 5GW 算力承诺，本质上是 AWS 对 Anthropic 议价能力增长的应激反应。

**收入从 $90 亿到 $300 亿只用了 4 个月**，验证了整个平台化战略的正确性。此刻，Anthropic 不再是一家 AI 创业公司，而是一个平台。

### 战略弧线总结

| 步骤 | 行动 | 目的 |
|------|------|------|
| 第一步 | 用算力换资本，用资本换分发管道 | 云合作，建立触达基础 |
| 第二步 | 用分发管道触达终端企业，用旗舰案例建立行业信任 | 直接客户，积累案例 |
| 第三步 | 用咨询网络解决落地壁垒，用行业方案提升客单价 | 咨询 + 垂直化，解决最后一公里 |
| 第四步 | 用数据层整合强化工作流锁定，用 MCP 生态占领界面 | 平台化，建立数据护城河 |
| 第五步 | 直接进入应用层，把原来依赖自己的 SaaS 公司变成竞争对手 | 颠覆，收割中间层利润 |

## 第五章：如果你是合作伙伴，应该如何思考

从这些案例中，可以提炼出每类合作伙伴需要面对的核心问题和决策框架。

### 如果你是云服务商

**核心问题**：如何防止 Anthropic 稀释你的独家优势？

Anthropic 的"三云全覆盖"策略让 AWS 从独家合作伙伴变成了三分之一。AWS 的应对方式是**追加投资 + 深化技术绑定**（Trainium 芯片联合研发）。

对于其他云服务商：没有投资关系的独家性很难维持，只能靠**数据主权**（本地化部署）、**行业合规**（特定行业的独家云合规认证）或**价格竞争**来维持差异化。

### 如果你是咨询公司

**核心问题**：你是在帮 Anthropic 培养客户，还是在建立自己的壁垒？

Anthropic 进入应用层（Cowork、垂直方案）的趋势不可逆——它最终会吃掉部分实施咨询市场。咨询公司的护城河只有两个：

- **行业 Domain Knowledge**：Anthropic 无法快速复制的监管和业务经验
- **客户关系**：企业信任的是 Deloitte，而不是 Anthropic

专注于这两个壁垒，而非技术实施本身。

### 如果你是数据 / SaaS 平台

**核心问题**：你的数据护城河够深到让 Anthropic 不敢绕开你吗？

Thomson Reuters 的教训：如果数据可以被替代，或者产品只是"模型 + 包装"，Anthropic 直接进入是时间问题。

**能活下来的是拥有不可复制数据资产的公司**：穆迪的 6 亿实体数据、Westlaw 的法律案例数据库。如果你只是在 Claude API 上包一层 UI，Anthropic 进入应用层只需要发布一个插件。

### 如果你是终端企业客户

**核心问题**：你的工作流锁定程度，是你能承受的吗？

NBIM、Banner Health 都是"全面押注 Claude"的极端案例——在当前阶段，这是合理的效率选择，但需要清醒地认识到迁移成本。

建议：
- **在工作流设计层面保持一定的抽象层**（通过 API 网关或中间件）
- **保持对多模型的评估能力**，避免单一供应商依赖
- **避免将内部数据资产直接绑定到特定供应商**
- 锁定可以接受，但要主动管理

## 结语

所有合作关系的最终指向是一个问题：**企业在"思考"的时候，调用的是谁的认知基础设施？**

当 NBIM 的分析师在 Claude 里查看投资组合，当 Banner Health 的医生用 BannerWise 看病历，当花旗的工程师用 Claude 写代码，当 AIG 的核保人用 Claude 审单——这些工作流的每一次运转，都是对 Claude 认知基础设施的一次调用，都是对 Anthropic 的一笔账单。

但更重要的是：每一次调用都是一次数据点，告诉 Anthropic 企业在实际工作中需要什么。这是一个任何竞争对手都无法通过购买数据集来复制的优势——它是实时的、在生产环境中的、有标注的真实需求信号。

**Anthropic 的商业模式本质：它在用"安全"换信任，用"信任"换渗透，用"渗透"换锁定，用"锁定"换定价权。**

所有的合作伙伴，无论是 AWS、Accenture、Snowflake 还是 NBIM，都是这个过程中的一个环节。没有任何一个环节是 Anthropic 的"利他"行为——但也没有任何一个环节是单方面剥削。这是一场精密设计的利益共同体建设。

问题是：当 Anthropic 最终完成平台化，成为不可替代的认知基础设施时，这些合作伙伴的议价能力会在哪里？

这个问题，现在没有人有答案。但 Thomson Reuters 的股票图表，已经提前给出了市场的预判。
